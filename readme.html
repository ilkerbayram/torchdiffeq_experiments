Experiments with torchdiffeq

This repository contains code demonstrating the use of torchdiffeq on a simple experiment (somewhat even simpler than the one in the original repo).

To get the figures below, run demo.py under ./code/ -- make sure you run the script from that directory.

Differentiating a Parametric ODE

Consider a one-dimensional nonlinear ODE of the form

ddtx(t)=fΘ(x(t)),

where fΘ:R→R is a parametric function that varies smoothly with respect to its parameters Θ (e.g., a neural network).

In practice, the function x can be computed numerically using an integrator like scipy's solve_ivp.

While the notation does not suggest it, notice that x(t) depends on Θ. Therefore, for a given initial value we can change the behavior of x(t) by varying Θ. Ignoring issues on existence, we can also talk about the derivative of x(t) with respect to Θ, for each value of t, namely

∂∂Θx(t).

torchdiffeq provides an efficient numerical procedure that relies on the adjoint method to obtain the derivative above.

torchdiffeq in Action

Let us now discover how we can use this derivative to approximate a nonlinear ODE.

Consider the following ODE :

x˙(t)=sin⁡(2πx(t)).

Notice that when x is a multiple of 0.5, the right hand side equals zero. These constitute the fixed points of the ODE.

However, some of these fixed points are stable, and some are unstable. Consider the phase portrait, shown below. The stable fixed points occur whenever the dynamics crosses the x-axis with a negative slope.

Specifically, when x=ϵ, the dynamics pushes x away from the fixed point at 0. This continues until x reache the next stable point on the right, at 0.5.

The converse occurs when we start from x=−ϵ. The point gets pushed away from 0 until it reaches the next stable fixed point at x=−0.5. These two solutions, corresponding to the two slightly different initial conditions are shown below.

Suppose now that we observe these two curves, and we would like to estimate f(⋅) in

x˙=f(x).

Let X denote the available data.

We will estimate f(⋅) using a simple neural network. Specifically, let fΘ(⋅) denote a multilayer perceptron with two hidden layers having 100 neurons each. We set the input and output sizes as 1, to match our problem. We also use tanh as our nonlinearity -- there is no activation at the last layer.

To set up a loss function,

  1. we extract a segment from the available data, of the form x(t0),…,x(tn).
  2. We then simulate xΘ(t0),…,xΘ(tn) the state dynamics are defined via xΘ˙=fΘ(xΘ), and xΘ(t0) is set to x(t0).
  3. Finally, using x and xΘ, we form the loss function L(Θ)=1n∑i=1n|x(ti)−xΘ(ti)|2.

Given the differentiable loss function, we obtain ∂ΘL(Θ) via torchdiffeq, and take steps so as to minimize the average of L(Θ) (averaged over the segments).

Note that we are free to form batches, which we do, to come up with less noisy gradients of the loss function.

Below are the results of this optimization procedure.

First, before starting the initialization, the MLP fΘ is an arbitrary function, far from sin⁡(2πx), as shown below.

But after training, we get a much closer function, as shown below. Notice that we provide data to the network only in the range x∈[−0.5,0.5]. Therefore, beyond that interval, we don't really expect fΘ to well-approximate the unknown non-linearity.
